🔍 노드별 상세 설명

노드타입입력출력LLM 호출
intake LLM user_input (text or dict) normalized_order_intent, trader_id, persona NL 입력 시 1회
policy 결정론 trader_id, persona persona_bounds, trader_principles (RAG) 없음
strategy LLM order_intent + trader_principles llm_proposal (action, size_usd, conviction) 1회
risk_agent ReAct llm_proposal + tools risk_tool_calls (thought + result) 최대 4회 (tool loop)
risk_gate 결정론 llm_proposal, persona_bounds risk_assessment, lifecycle_status 없음
evaluator LLM 전체 state evaluator_review (user_message) 1회

🟠 risk_agent ReAct 루프 상세

sequenceDiagram participant G as risk_agent participant LLM as Gemini 2.5 participant T as Tool Registry G->>LLM: 주문 정보 + 도구 목록 전달 LLM-->>G: FunctionCall(get_balance, {asset: USDT}) G->>T: dispatch("get_balance", {asset: USDT}) T-->>G: {free: 5000.0, locked: 0.0} G->>LLM: 함수 결과 전달 (turn 2) LLM-->>G: FunctionCall(get_volatility, {symbol: BTCUSDT}) G->>T: dispatch("get_volatility", {symbol: BTCUSDT}) T-->>G: {atr_pct: 0.03, verdict: normal} G->>LLM: 함수 결과 전달 (turn 3) LLM-->>G: is_final=True (텍스트 응답) G->>G: risk_tool_calls 저장 → risk_gate 전달
MAX_TOOL_CALLS = 4 — LLM이 계속 도구를 호출해도 최대 4번에서 강제 종료.
thought 필드 — LLM이 도구 호출 전 내부 사고 과정을 기록. LangSmith에서 확인 가능.
risk_gate가 최종 결정 — ReAct 결과와 관계없이 결정론 7-체크가 최종 verdict를 결정.
